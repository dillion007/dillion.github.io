第一篇：处女作

发表于 2017-03-12

缘起

一直想着，什么时候开个博客，写点东西。如果在csdn、简书等上面开一个，感觉不符合一个程序猿的逼格。昨天晚上有点时间，于是就在搜索引擎上查找怎样构建一个博客。

技术框架选择

作为一个博客，选择一个静态网页来做，是一个合适的方案，简单又好用。主流的静态网页的框架有Jekyll、Octopress、Hexo等11种。作为一个初学者，最好的选择肯定是一个轻量级的框架。任何东西，不要一开始就选择一个功能很强大，但是用起来麻烦的东西，因为这会消耗掉一个人最初的热情。只有能先用起来，才会发现是否会有不足，然后再考虑是否要换框架。看到了简书上的一篇《5分钟 搭建免费个人博客》，就决定用Hexo。呵呵，是不是有点随意。

动手搭建

在搜索引擎上找了4、5篇文章，大致都看了一下，最后按照《史上最详细的Hexo博客搭建图文教程》开始搭建。期间遇到了以下问题，使用搜索引擎一一解决。

GitHub Pages 的仓库名

仓库名必须为”username.github.io” ，否则最后就没办法用 https://username.github.io 进行访问。实际上，页面会变成 https://github.com/anotherName.github.io 。如果名字取错了，可以点击到repositories去rename，或者删除了重新再建一个仓库。

Hexo安装

按照教程进行安装，最后localhost:4000，无法访问。试了几次之后，还是不行，于是进Hexo的官方网站，按照上面的步骤，试了几次也是不行。意识到步骤上应该没有问题，作为一个成熟的框架，不可能只有我一个人遇到问题，万能的Google一定能找到答案。只要换一个端口就能解决，Hexo s -p 5000，搞定！

最后

推送到GitHub，更新文章，这一步倒是没有遇到什么问题，作为一个经常使用git的程序猿，毫无压力。
